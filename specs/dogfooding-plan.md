# План: Dogfooding Agent Console

## Цель
Использовать Agent Console для разработки самой Agent Console.

---

## Шаг 1 — Минимальный рабочий цикл

Один агент (Developer) + человек как ревьюер.

**Цикл:**
```
Задача → Developer пишет код → Diff в чате → Approve/Reject → Merge
```

**Что нужно сделать:**
- [ ] Проверить что agent execution работает end-to-end
- [ ] Показать изменения агента в чате (diff preview)
- [ ] Approve/Reject кнопки → merge или откат
- [ ] Git auth через product secrets (не глобальный GITHUB_TOKEN)
- [ ] Терминал в UI (xterm.js + WebSocket) — запуск команд из интерфейса

---

## Шаг 2 — Расширение команды агентов

Когда цикл из шага 1 работает, добавляем агентов по одному.
Каждый новый агент проверяется на реальной задаче разработки платформы.

```
Planner   → декомпозиция задачи на подзадачи
Developer → пишет код (уже есть)
Reviewer  → автоматический code review (уже есть)
Tester    → запуск тестов, проверка что не сломал
```

**Что нужно сделать:**
- [ ] Planner agent — принимает задачу, разбивает на подзадачи
- [ ] Tester agent — запускает тесты, отчитывается о результатах
- [ ] Настроить workflow: Planner → Developer → Reviewer → Tester
- [ ] UI для отображения цепочки выполнения (какой агент сейчас работает)

---

## Шаг 3 — Spec-Driven Development

Полная цепочка от спецификации до готового кода.

```
Spec Agent → Architect → Planner → Developer → Reviewer → Tester
```

**Что нужно сделать:**
- [ ] Spec Agent — генерирует/обновляет SPEC.md из задачи
- [ ] Architect Agent — проектирует решение, обновляет ARCHITECTURE.md
- [ ] Интеграция с .ai/ папкой — агенты читают и обновляют спецификации
- [ ] HITL gate между этапами (человек может вмешаться на любом шаге)

---

## Текущие blockers для шага 1
1. **Agent execution** — нужно проверить текущее состояние, найти что сломано
2. **Diff в чате** — агент делает изменения, но пользователь не видит что именно
3. **Approve/Reject** — нет кнопок в UI для одобрения MR
4. **Git auth** — push не работает без токена в контейнере
5. **Терминал** — нет способа запустить команды из UI

## Приоритет
Сначала — шаг 1. Без рабочего цикла остальное бессмысленно.
